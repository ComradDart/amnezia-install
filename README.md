# amnezia-install

Интерактивный скрипт первичной настройки VPS под **AmneziaWG** с веб-панелью
[wg-easy](https://github.com/wg-easy/wg-easy), спрятанной за Nginx по секретному пути.

Один запуск превращает чистый Ubuntu/Debian-сервер в готовый VPN-узел с базовой
защитой: новый sudo-пользователь, ужесточённый SSH, файрвол, fail2ban, модуль ядра
AmneziaWG, панель управления в Docker и сайт-заглушка с TLS.

## Что делает скрипт

1. **Безопасность.** Создаёт нового пользователя вместо root, переносит ему SSH-ключи,
   отключает root-вход и (при наличии ключей) парольную аутентификацию SSH, настраивает
   `ufw` (deny по умолчанию, открыты только нужные порты) и `fail2ban`.
2. **AmneziaWG.** Ставит модуль ядра `amneziawg` из официального PPA `amnezia/ppa`
   (на Debian репозиторий подключается вручную с проверкой ключа), включает автозагрузку.
3. **Панель wg-easy.** Разворачивает официальный образ `ghcr.io/wg-easy/wg-easy:15`
   в Docker в режиме AmneziaWG (`EXPERIMENTAL_AWG=true`). Панель вообще не публикуется
   на хост — Nginx обращается к ней по имени сервиса внутри docker-сети.
4. **Nginx (в Docker) + заглушка + секретный путь.** Nginx и wg-easy поднимаются
   одним `docker-compose` в общей сети `wg`; наружу торчат только 80/443 и порт WG.
   На 80/443 по умолчанию отдаётся статическая заглушка, а доступ к панели открывается
   только по случайно сгенерированному пути (`openssl rand -hex 12`).
5. **TLS-сертификат** — на выбор: самоподписанный, Let's Encrypt (HTTP-01) или
   Let's Encrypt через Cloudflare (DNS-01).

## Роли и двойной прыжок для Telegram (double-hop)

В начале скрипт спрашивает **роль сервера**:

- **`outbound`** — сервер за рубежом (точка выхода). Это всё перечисленное выше; плюс
  опционально поднимается **telemt** (выход к Telegram) и его веб-панель. telemt и панель
  делят сетевой namespace с wg-easy, поэтому telemt слушает **строго на IP внутри
  туннеля** AmneziaWG и недоступен снаружи напрямую.
- **`inbound`** — сервер в РФ (точка входа). Ставит **HAProxy** (SNI-роутер на :443),
  **nginx** с легитимным сайтом-заглушкой и **AmneziaWG-клиента**, подключённого к
  outbound. Трафик Telegram (по маске SNI) уходит в туннель на telemt, всё остальное —
  на легит-сайт.

Схема (по мотивам [telemt VPS_DOUBLE_HOP](https://github.com/telemt/telemt/blob/main/docs/Setup_examples/VPS_DOUBLE_HOP.ru.md)):

```
Telegram ──443──> INBOUND (РФ)                         OUTBOUND (за рубежом)
                  HAProxy :443 ─SNI=маска─AmneziaWG──>  telemt (на WG-IP туннеля)
                           └─иначе──> nginx (легит-сайт)  wg-easy + nginx + панели
```

**Порядок развёртывания:**

1. Поднимите **outbound** (`роль = outbound`, включите telemt). В конце скрипт напечатает
   WG-IP telemt, порт, маску fake-TLS, секрет, готовую `tg://`-ссылку и адрес панели telemt.
2. В панели wg-easy на outbound создайте клиента (**New Client → скачать `.conf`**).
3. Поднимите **inbound** (`роль = inbound`): вставьте этот `.conf` и укажите WG-IP/порт
   telemt и маску из печати outbound.
4. Подключитесь по `tg://`-ссылке — Telegram пойдёт двойным прыжком: РФ → туннель → выход.

Без Telegram всё работает как обычный VPN — просто выберите роль `outbound` и откажитесь
от telemt.

## Требования

- Чистый VPS с **Ubuntu 22.04 / 24.04** или **Debian 11 / 12**
- Доступ root (или sudo)
- Для Let's Encrypt — домен, направленный на сервер; для Cloudflare DNS-01 — API-токен
  Cloudflare (шаблон «Edit zone DNS»)

## Запуск

Зайдите на сервер по SSH под root и скачайте скрипт прямо с GitHub:

```bash
# через curl
curl -fsSLO https://raw.githubusercontent.com/ComradDart/amnezia-install/main/install.sh
bash install.sh
```

```bash
# либо через wget
wget https://raw.githubusercontent.com/ComradDart/amnezia-install/main/install.sh
bash install.sh
```

> **Не запускайте через `curl ... | bash`.** Скрипт интерактивный (`read -p`), а при
> запуске из пайпа stdin занят загрузкой и ответы ввести не получится. Сначала
> скачайте файл, потом запустите `bash install.sh`.

Скрипт интерактивно спросит:

| Запрос | По умолчанию |
|---|---|
| Имя нового пользователя (вместо root) | `vpnadmin` |
| Порт AmneziaWG (UDP) | `51820` |
| Публичный IP или домен сервера | — |
| Способ получения TLS-сертификата | `1` (самоподписанный) |
| E-mail для Let's Encrypt (для вариантов 2/3) | `admin@<домен>` |
| API-токен Cloudflare (для варианта 3) | — |

В конце выводятся URL панели, логин и пароль администратора.

## Как устроен доступ к панели

wg-easy — это SPA с абсолютными путями (`/api`, `/_nuxt`), поэтому проксирование
подпути сломало бы интерфейс. Вместо этого используется схема на cookie:

- заход на `https://<домен>/<секретный_путь>` ставит браузеру `HttpOnly`-cookie
  на 12 часов и редиректит на `/`;
- Nginx проксирует на панель только запросы с верной cookie;
- всем остальным запросам на 80 и 443 (на любом пути) отдаётся заглушка.

Снаружи, не зная секретного пути, панель не видна. Секрет и пароль сохраняются в
`/root/panel_path.txt`.

## Безопасность по умолчанию

- root-вход по SSH запрещён; парольная аутентификация отключается **только если**
  у пользователя есть SSH-ключ (защита от блокировки доступа);
- ufw открывает лишь: текущий порт SSH (по умолчанию `22/tcp`), `80/tcp`, `443/tcp`
  и выбранный UDP-порт AmneziaWG;
- `fail2ban` с jail `sshd` (backend systemd — работает и на Debian 12);
- панель доступна снаружи только по TLS и только по секретному пути.

> **Не закрывайте текущую SSH-сессию** до того, как проверите вход новым
> пользователем в отдельном окне — root-доступ по SSH будет уже отключён.

## Идемпотентность и логи

Скрипт можно запускать повторно: каждый шаг проверяет фактическое состояние
(установленные пакеты, наличие пользователя, загруженный модуль, содержимое
конфигов) и пропускает уже выполненное. Сгенерированные секреты и ответы
сохраняются в `/var/lib/vps_setup/` и переиспользуются.

Все действия с временными метками пишутся в `/var/log/vps_setup.log`.
